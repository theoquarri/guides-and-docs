Onboarding Guide

Connect Shopify to Quarri

Set up a Shopify custom app so Quarri can pull your orders, customers, products, and inventory into a single source of truth. The whole process takes about 10 minutes.

Before You Start
What You'll Need
A quick checklist before we begin — gather these and the rest of the setup is paint-by-numbers.

Checklist

• Owner-level access to your Shopify store (you need to be able to create custom apps).

• Your Shopify store ID — the prefix of your admin URL. For yourstore.myshopify.com, the store ID is yourstore.

• About 10 minutes.

Part 1 — Shopify
Create the Custom App in Shopify
Quarri pulls data using a Shopify custom app you create inside your own store. This lives in your Shopify Dev Dashboard and is yours to manage and revoke at any time.
1
Open Develop Apps
In your Shopify admin, go to Settings → Apps and sales channels → Apps, then click Develop apps in the top-right corner.
2
Build apps in Dev Dashboard
Click Build apps in Dev Dashboard. This is Shopify's newer app management environment — it's required for new custom apps because the legacy custom-apps path has been deprecated.
3
Create the app
In the Dev Dashboard, click Create app in the top-right corner.
4
Name your app
Choose the Start from Dev Dashboard option (right-hand side, not the Shopify CLI option). Name the app something memorable like Quarri and click Create.
5
Configure the app URLs
Open the new app's Configuration tab. Shopify requires a few URL fields, but for a data integration like Quarri none of them are actually used at runtime — they're just required form fields. Fill them in as follows so the app is self-documenting later:
FieldValueWhy
App URL https://quarri.ai Required field, but never traversed. Using Quarri's URL makes the app self-documenting.
Preferences URL (leave blank) Optional. Only used by embedded apps with settings screens.
Redirect URLs https://quarri.ai Required, but the client-credentials grant Quarri uses never redirects the user.
Embed app in Shopify admin Off This isn't an embedded app.
Use legacy install flow Off Stay on the modern flow.
Webhooks API version Latest (e.g. 2026-04) Default is fine. Quarri reads from the Admin API directly.
6
Add the API scopes
In the Access scopes section of the Configuration tab, add each of the following read-only scopes to the Scopes field. These are the permissions Quarri needs to read your store data — and only the ones it needs. Quarri never writes anything back to Shopify.
  • read_assigned_fulfillment_orders
  • read_checkouts
  • read_fulfillments
  • read_inventory
  • read_locales
  • read_locations
  • read_marketing_events
  • read_marketplace_fulfillment_orders
  • read_merchant_managed_fulfillment_orders
  • read_orders
  • read_all_orders
  • read_payment_terms
  • read_products
  • read_publications
  • read_own_subscription_contracts

Why read_all_orders?

By default Shopify only exposes the last 60 days of orders. read_all_orders unlocks the full history so Quarri can build accurate year-over-year reporting.

Leave the Optional scopes field blank. Click Save at the bottom of the page.
7
Install the app to your store
Back on the app's Overview page, click Install app in the top-right. You'll be asked to confirm the scopes — review them and accept.
8
Copy your credentials
Once the app is installed, open the API credentials tab. You'll see three values to send to Quarri:
  • Client ID
  • Client secret (reveal it and copy — Shopify will let you view it again, but treat it like a password)
  • Store ID — the prefix of your *.myshopify.com URL

Treat the Client Secret like a password

Anyone with these two values can read everything your scopes allow. Send them to Quarri via a secure channel (1Password share link, encrypted email, or our secure intake form) — not in a plain email or Slack message.

Part 2 — Quarri
Hand the Credentials to Quarri
Once Quarri has your three values, we'll wire up the pipeline on our side. You don't need to touch the connector configuration yourself.
1
Send us the credentials
Share your Client ID, Client secret, and Store ID with your Quarri contact. We'll let you know once the connector is live — typically within the same business day.
2
First sync
Quarri does an initial historical pull of your store (orders, customers, products, inventory, locations) and then keeps it incrementally up to date. The first full sync can take a couple of hours for larger stores; ongoing syncs are near-real-time.
3
Start querying
Once the first sync is complete, you can ask questions of your Shopify data in plain English through Claude — alongside data from your other systems. Example queries:
  • "What were our top 10 products by revenue last month?"
  • "Show me customers who bought twice in the last 90 days but haven't returned."
  • "Compare AOV between TikTok Shop and the .com store, week by week."
FAQ
Common Questions
Is this read-only?
Yes. Every scope above starts with read_. Quarri cannot create, modify, or delete anything in your store.
How do I revoke access?
The app lives in your Shopify Dev Dashboard under your control. You can uninstall it from Settings → Apps and sales channels at any time, or rotate the Client secret to invalidate Quarri's access without uninstalling. Quarri's pipeline will stop receiving data the moment you do either.
Will this slow down my store?
No. Quarri queries Shopify's Admin API on its own connection — it has no effect on your storefront performance.
What plans does this work on?
All Shopify plans. The modern client-credentials flow Quarri uses works on Basic, Shopify, Advanced, Plus, and Grow tiers without limitation.
